### Scenario 5: Task Scheduler System

### User Story:-

    As a system administrator, I want to schedule and prioritize tasks so that system performance is optimized.

### Problem Statement:-

    Design a task scheduling system.

### Requirements:-

    Use Queue<string> for task execution order

    Use Stack<string> for undoing the last executed task

    Use List<string> for storing all tasks

    Use SortedDictionary<int, string> for priority-based tasks

    Use HashSet<string> to ensure no duplicate tasks

### Expected Outcome:-

    Execute tasks in order

    Undo the last operation

    Maintain priority-based scheduling

### Question in breif

    Question 5: Design a Task Scheduler System where you use a Queue for task execution order, a Stack for undoing the 
    last executed task, a List to store all tasks, a SortedDictionary for priority-based tasks, and a HashSet to prevent 
    duplicate tasks. The system should execute tasks in sequence, support undo functionality, and handle priority-based 
    scheduling.
